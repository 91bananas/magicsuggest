MagicSuggest

Auto-suggest combo with bootstrap theme using jQuery

Download this project as a .zip file Download this project as a tar.gz file

I needed to create a highly flexible auto suggest combo box in the line of Bootstrap themes.
All the combos I've seen didn't have all the options I needed so I went and rolled my own.
You're free to do whatever you wish with it. (permissive MIT licence)

This combo was generated with the default options.

With no configuration options set, you have the following behaviour:

- the side trigger is visible and the selected items appear within the combo itself.
- suggestions will appear as the user starts typing
- the user can toggle the suggestions using the down arrow key
- the user can delete selected items by using backspace
- the user is free to enter new input using the enter key after having entered text

There are many configuration options that allow you to customize the component.

Installation Guide

1. Download the latest version of the project using one of the two links above.

2. Copy the content of the bin folder into your project to use the minified version.

3. Add the following lines in your HEAD contents

<link rel="stylesheet" href="magicsuggest-min.css">
<script type="text/javascript" src="jquery-1.8.3-min.js"></script>
<script type="text/javascript" src="class-0.0.2-min.js"></script>
<script type="text/javascript" src="magicsuggest-min.js"></script>

4. To transform any DOM element into a MagicSuggest combo, simply include this after jQuery has been initialized: 

new MagicSuggest({options});

Configuration Properties

allowFreeEntries
boolean
Restricts or allows the user to validate typed entries.
Defaults to true View an example
cls
string
A custom CSS class to apply to the field's underlying element.
Defaults to '' View an example
data
array / string
JSON Data source used to populate the combo box. 3 options are available here:

No Data Source (default)
When left null, the combo box will not suggest anything. It can still enable the user to enter multiple entries if allowFreeEntries is set to true (default).

Static Source
You can pass an array of JSON objects, an array of strings or even a single CSV string as the data source.
For ex. data: [{id:0,name:"Paris"}, {id: 1, name: "New York"}]

Url
You can pass the url from which the component will fetch its JSON data.
Data will be fetched using a POST ajax request that will include the entered text as 'query' parameter. The results fetched from the server can be:
- an array of JSON objects (ex: [{id:...,name:...},{...}])
- a string containing an array of JSON objects ready to be parsed (ex: "[{id:...,name:...},{...}]")
- a JSON object whose data will be contained in the results property (ex: {results: [{id:...,name:...},{...}]

Defaults to null View an example
disabled
boolean
Start the component in a disabled state.
Defaults to false. View an example
displayField
string
name of JSON object property displayed in the combo list.
Defaults to name
editable
boolean
Set to false if you only want mouse interaction. In that case the combo will automatically expand on focus.
Defaults to true View an example
emptyText
string
The default placeholder text when nothing has been entered.
Defaults to 'Type or click here' or just 'Click here' if not editable. View an example
emptyTextCls
string
A custom CSS class to style the empty text.
Defaults to 'ms-empty-text' View an example
expanded
boolean
Set starting state for combo.
Defaults to false View an example
expandOnFocus
boolean
Automatically expands combo on focus.
Defaults to false View an example
hideTrigger
boolean
Set to true to hide the trigger on the right.
Defaults to false View an example
highlight
boolean
Set to true to highlight search input within displayed suggestions.
Defaults to true View an example
invalidCls
string
The class that is applied to show that the field is invalid.
Defaults to ms-ctn-invalid
matchCase
boolean
Set to true to filter data results according to case. Useless if the data is fetched remotely.
Defaults to false View an example
maxDropHeight
integer
Once expanded, the combo's height will take as much room as the # of available results. In case there are too many results displayed, this will fix the drop down height.
Defaults to 300 px View an example
maxResults
integer
The maximum number of results displayed in the combo drop down at once. Set to false to remove the limit.
Defaults to 10 View an example
maxSelection
integer
The maximum number of items the user can select if multiple selection is allowed. Set to false to remove the limit.
Defaults to 10 View an example
method
string
Method used by the ajax request.
Defaults to 'POST'
minChars
integer
The minimum number of characters the user must type before the combo expands and offers suggestions. You can set this to 0 if you want the drop down to expand as soon as the component gains focus.
Defaults to 0 when data is null or set to local storage, 2 otherwise. View an example
preselectSingleSuggestion
boolean
If a single suggestion comes out, it is preselected.
Defaults to true View an example
renderTo
element
If set, the combobox will be rendered within the given element.
Defaults to null
required
string
Whether or not this field should be required.
Defaults to false View an example
resultAsString
boolean
Set to true to render selection as comma separated string
Defaults to false View an example
selectionCls
string
A custom CSS class to add to a selected item.
Defaults to '' View an example
selectionPosition
string
Where the selected items will be displayed. Only 'right', 'bottom' and 'inner' (within the combo) are valid values
Defaults to 'inner' View an example
selectionStacked
boolean
Set to true to stack the selectioned items when positioned on the bottom. Requires the selectionPosition to be set to 'bottom'
Defaults to false View an example
sortDir
string
Direction used for sorting. Only 'asc' and 'desc' are valid values.
Defaults to 'asc' View an example
sortOrder
string
name of JSON object property for local result sorting. Leave null if you do not wish the results to be ordered or if they are already ordered remotely.
Defaults to null View an example
useTabKey
boolean
If set to true, tab won't blur the component but will be registered as the ENTER key.
Defaults to false View an example
useCommaKey
boolean
If set to true, using comma will validate the user's choice (same as ENTER).
Defaults to true View an example
useZebraStyle
boolean
Determines whether or not the results will be displayed with a zebra table style.
Defaults to true View an example
value

initial value for the field.
Defaults to null View an example

name of JSON object property that represents its underlying value.
