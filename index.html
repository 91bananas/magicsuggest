MagicSuggest

Auto-suggest combo with bootstrap theme using jQuery

Download this project as a .zip file Download this project as a tar.gz file

I needed to create a highly flexible auto suggest combo box in the line of Bootstrap themes.
All the combos I've seen didn't have all the options I needed so I went and rolled my own.
You're free to do whatever you wish with it. (permissive MIT licence)

This combo was generated with the default options.

With no configuration options set, you have the following behaviour:

- the side trigger is visible and the selected items appear within the combo itself.
- suggestions will appear as the user starts typing
- the user can toggle the suggestions using arrow keys
- the user can delete selected items by using backspace
- the user is free to enter new input using the enter key after having entered text

There are many configuration options that allow you to customize the component.

Installation Guide

1. Download the latest version of the project using one of the two links above.

2. Copy the content of the bin folder into your project to use the minified version.

3. Add the following lines in your HEAD contents

<link rel="stylesheet" href="magicsuggest-x.y.z-min.css">
<script type="text/javascript" src="jquery-1.8.3-min.js"></script>
<script type="text/javascript" src="magicsuggest-x.y.z-min.js"></script>

4. To transform any DOM element into a MagicSuggest combo, simply include this after jQuery has been initialized: 

$(el).magicSuggest({options});

Configuration Properties

Notice for 1.2.5+ users

Note that as of version 1.2.5, the component's configuration is also generated from the original DOM element container.

The following 2 blocks of code will generate the same component:

<div id="ms1" name="colors" data="red,blue,green" value="red"></div>
<script>
    $('#ms1').magicSuggest({});
</script>
 
<div id="ms1"></div>
<script>
    $('#ms1').magicSuggest({
        id: 'ms1',
        name: 'colors',
        data: 'red,blue,green',
        value: 'red'
    });
</script>

This allows you to setup your components directly in your DOM and eventually perform a bulk component creation by using a broader jQuery selector (on a class for example)

Config options
allowFreeEntries
boolean
Restricts or allows the user to validate typed entries.
Defaults to true View an example
cls
string
A custom CSS class to apply to the field's underlying element.
Defaults to '' View an example
data
array / string
JSON Data source used to populate the combo box. 3 options are available here:

No Data Source (default)
When left null, the combo box will not suggest anything. It can still enable the user to enter multiple entries if allowFreeEntries is set to true (default).

Static Source
You can pass an array of JSON objects, an array of strings or even a single CSV string as the data source.
For ex. data: [{id:0,name:"Paris"}, {id: 1, name: "New York"}]

Url
You can pass the url from which the component will fetch its JSON data.
Data will be fetched using a POST ajax request that will include the entered text as 'query' parameter. The results fetched from the server can be:
- an array of JSON objects (ex: [{id:...,name:...},{...}])
- a string containing an array of JSON objects ready to be parsed (ex: "[{id:...,name:...},{...}]")
- a JSON object whose data will be contained in the results property (ex: {results: [{id:...,name:...},{...}]

Defaults to null View an example

Frequently Asked Questions

My combo is within my form but I don't retrieve any data in the backend?

In order for MagicSuggest component to register for a form component, one must add a name property to its configuration or original DOM container.

The values retrieved in the back-end will be an array of IDs

$('#ms-tpl').magicSuggest({
    data: [{id:1, name: 'red'}, {id: 2, name: 'blue'}, {id: 3, name: 'green'}],
    name: 'color'
});

Selecting red and blue will result in sending [1,2] in the POST parameter called color.

How do I use events?

You first need to create a variable that will store your magicSuggest object. You can then use the following:

var ms = $('#ms').magicSuggest({
   data: 'red,blue,green'
});
$(ms).on('selectionchange', function(event, combo, selection){
     alert(selection);
});

The value property doesn't work when working with remote data?

This is a common case when you are trying to set initial values to the combo while its data hasn't been loaded yet.

What you need to do is to perform an initial load with special parameters in order to validate the data in the backend.

Here is an example with PHP:

data.php:

<?php

// this would usually be fetched through a database using a specific DAO.
$colors = array(
    array('id' => 0, 'name' => 'red'),
    array('id' => 1, 'name' => 'blue'),
    array('id' => 2, 'name' => 'green')
);

/**
 * Retrieve colors according to given name query
 * $query - string : user query
 */
function getColorsByName($query){
    global $colors;
    if(empty($query)) return $colors;
    $matches = array();
    for($i = 0; $i < count($colors); $i++){
        if(strpos($colors[$i]['name'], $query) > -1){
            $matches[] = $colors[$i];
        }
    }
    return $matches;
}

/**
 * Retrieve colors according to given ids
 * $arrIds - array : array of color ids.
 */
function getColorsbyId($arrIds){
    global $colors;
    $matches = array();
    for($i = 0; $i < count($colors); $i++){
        if(array_search($colors[$i]['id'], $arrIds) !== false){
            $matches[] = $colors[$i];
        }
    }
    return $matches;
}

if(isset($_POST['init'])){
    echo json_encode(getColorsbyId($_POST['cols']));
} else {
    echo json_encode(getColorsByName($_POST['query']));
}

?>

index.html javascript part:

$(document).ready(function() {
    var ms = $('#ms').magicSuggest({
        data: 'data.php',
        dataUrlParams: {
            init: true,
            cols: [2,1]
        }
    });

    $(ms).on('load', function(){
        if(this._dataSet === undefined){
            // we get here the first time the combo has loaded and gone through
            // the first if case in tag-autocomplete-service.php: the combo has been
            // populated with the default values only.
            this._dataSet = true;
            ms.setValue([2,1]);
            ms.setDataUrlParams({});
        }
    });
});
